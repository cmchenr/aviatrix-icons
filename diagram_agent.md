## Icon Selection Guidelines

### By Cloud Provider
- Use the appropriate cloud provider folder (aws, azure, gcp, oracle, alibaba) for provider-specific resources
- Use `edge` for on-premises, hybrid, or edge deployments

### By Resource Type
- **VPC/VNet**: Virtual private network container (`vpc`, `vnet`)
- **Subnet**: Network segment within a VPC
- **Gateway**: Aviatrix-managed gateway
- **Transit Gateway**: Hub gateway for transit architecture
- **Spoke Gateway**: Spoke gateway connecting to transit
- **Transit VPC/VNet**: VPC containing transit gateway
- **Spoke VPC/VNet**: VPC containing spoke gateway
- **VM**: Compute instance / virtual machine
- **Unmanaged**: Resources not managed by Aviatrix (grayed out styling)

### Aviatrix-Specific
- **FireNet**: Security inspection insertion
- **NAT**: Network address translation (AWS)
- **TGW**: AWS Transit Gateway integration

### Multi-Cloud
- Use `misc/multi-cloud` for diagrams spanning multiple providers
