# Contributing to Aviatrix Icons

Thank you for your interest in contributing to the Aviatrix Icons pack!

## How to Contribute

### Reporting Issues

If you find missing icons, incorrect assets, or have suggestions:

1. Check existing issues to avoid duplicates
2. Open a new issue with a clear description
3. Include cloud provider and resource type if applicable

### Submitting Icons

When submitting new or updated icons:

1. **Fork** this repository
2. **Create a branch** for your changes
3. **Add icons** following the guidelines below
4. **Submit a pull request** with a clear description

### Icon Guidelines

#### File Format
- SVG format only
- Optimized for web use (minimal file size)
- Viewbox should be consistent within each category

#### Naming Convention
- Use kebab-case: `transit-gateway.svg`, `unmanaged-vpc.svg`
- Be descriptive but concise
- Follow existing patterns in each directory

#### Directory Structure
Place icons in the appropriate directory:
- `icons/aviatrix/logo/` - Aviatrix brand assets
- `icons/aws/` - Amazon Web Services
- `icons/azure/` - Microsoft Azure
- `icons/gcp/` - Google Cloud Platform
- `icons/oracle/` - Oracle Cloud
- `icons/alibaba/` - Alibaba Cloud
- `icons/edge/` - Edge/Hybrid Cloud
- `icons/misc/` - Partner integrations and utilities

### Code of Conduct

- Be respectful and constructive
- Focus on the icons and their quality
- Help maintain consistency across the pack

## Questions?

Open an issue or reach out to the Aviatrix team.
